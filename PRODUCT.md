# Product

## Register

product

## Users

Developers who run coding agents across one or more repositories and need to stay oriented while agents work. They use Panes to move between conversations, terminal sessions, diffs, approvals, files, and Git state without losing the context of the current workspace.

## Product Purpose

Panes is a local first desktop cockpit for AI assisted coding. It gives developers one place to run multiple agent providers, review their work, make targeted edits, operate Git, and keep an auditable view of what happened. Success means the interface remains legible and calm while several tools, repositories, and sessions are active.

## Brand Personality

Precise, calm, and capable. The product should feel considered and technically credible, with the restraint of Cursor, Vercel, and OpenAI product surfaces. Terminal Mint identifies active and connected states without becoming decoration.

## Anti-references

Avoid neobrutalism, oversized marketing typography inside the product, generic dashboard cards, excessive borders, gratuitous glow and gradients, eyebrow labels, decorative motion, flat undifferentiated dark surfaces, and unfamiliar interaction patterns that interrupt coding flow.

## Design Principles

1. Keep the current task visually dominant while surrounding tools remain easy to reach.
2. Use surface depth and spacing to explain hierarchy before adding borders or labels.
3. Reserve brand color for action, focus, selection, and connection state.
4. Preserve familiar desktop and developer tool conventions.
5. Keep dense workflows readable through consistent typography and restrained contrast.

## Accessibility & Inclusion

Maintain WCAG AA text contrast, visible keyboard focus, semantic status colors that do not rely on hue alone, and reduced motion alternatives. Dark and light themes should preserve the same information hierarchy and interaction states.

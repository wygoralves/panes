# Product

## Register

product

## Users

Software developers who use external coding agents across one or more local repositories. They work in long-running desktop sessions and need to move between agent conversations, terminals, files, git state, approvals, and workspace setup without losing context or control.

## Product Purpose

Panes is a local-first desktop cockpit for AI-assisted coding. It brings agent chat, terminal sessions, git workflows, lightweight file editing, approvals, and audit trails into one application while leaving model execution to external tools. Success means developers can understand what each agent changed, control what it may do, and manage several repositories without relying on a CLI-only workflow.

## Brand Personality

Precise, calm, and capable. Panes should feel like a dependable technical instrument: compact enough for sustained use, clear about system state, and direct in its language.

## Anti-references

Avoid generic SaaS dashboards, decorative analytics, gratuitous gradients or glow, thick colored card stripes, oversized rounded cards, glass effects used as decoration, and interfaces that disguise important permissions or runtime state. Do not mimic a full IDE or overwhelm users with every advanced control at once.

## Design Principles

1. Keep context visible. Show scope, workspace, repository, and runtime state near the controls they affect.
2. Put common actions first. Keep advanced setup available through progressive disclosure.
3. Make consequences legible. Explain permissions, power behavior, integrations, and destructive actions in concrete language.
4. Preserve flow. Support dense, keyboard-friendly operation without forcing users through unnecessary dialogs.
5. Reuse one interaction vocabulary across chat, terminal, git, editor, and settings surfaces.

## Accessibility & Inclusion

Target WCAG 2.2 AA contrast and keyboard access for all controls. Provide visible focus states, text labels for icon-only actions, reduced-motion behavior, non-color status cues, and usable layouts at narrow desktop widths and increased text scaling. Keep English and Brazilian Portuguese resources aligned for every user-facing change.
